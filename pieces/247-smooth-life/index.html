Smooth Life — The Creature That Has No Name

Lenia and SmoothLife

SmoothLife, introduced by Stephan Rafler in 2011, is a continuous-space and continuous-state generalization of Conway's Game of Life. Where Life operates on a discrete grid with binary cell states, SmoothLife uses floating-point states in [0,1] spread across a continuous toroidal plane. The result is an ecology of slow, undulating forms — creatures that have no name, drifting and dividing like organisms filmed through a microscope.

The Neighborhood Rule

For each cell position, two averages are computed: n (inner density, disk of radius ri = 3) and m (outer density, annulus ri < r < ro = 6). The birth/survival function σ₂(n, birth_lo, birth_hi) maps these to a target state, implemented as a logistic sigmoid — giving the rule the same qualitative life/death character as Conway's rules, but infinitely smooth rather than hard-threshold.

Logistic Sigmoid Transition

The transition steepness α controls how sharp the birth/death boundary is. The four parameters (b₁ = 0.278, b₂ = 0.365, d₁ = 0.267, d₂ = 0.445) were found by Rafler to produce self-organising, traveling structures. The final state update next = clamp(2s − 1 + 0.9 × cur, 0, 1) adds damping so structures decay and re-form rather than instantly going dark or fully lit.

Color Mapping

State 0 maps to near-black (#050508), state 0.5 to deep violet (#7b3fff), and state 1.0 to bright cyan/white (#00e8d6 blending to white). A 256-entry LUT is precomputed once and applied per-pixel to minimize per-frame cost.

Performance

The simulation runs a naive O(W×H×r²) convolution over a small grid (default 128×128), scaled up by CSS image-rendering: pixelated to fill the canvas at ~4× zoom. This is sufficient for 8–20 steps/second on any modern device without WebGL.
